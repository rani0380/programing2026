01

하드웨어 구성

컴퓨터는 입력장치, 중앙처리장치, 기억장치, 출력장치로 구성됩니다. CPU는 제어장치, 연산장치, 레지스터를 중심으로 명령을 해석하고 처리합니다.

구성 요소역할
제어장치명령을 해석하고 장치 동작을 지시합니다.
연산장치산술 연산과 논리 연산을 수행합니다.
레지스터CPU 내부의 고속 임시 기억장치입니다.
02

기억장치

주기억장치는 현재 실행 중인 프로그램과 데이터를 저장하고, 보조기억장치는 장기 보관에 사용됩니다. 캐시는 CPU와 주기억장치 사이의 속도 차이를 줄입니다.

  • RAM: 읽기와 쓰기가 가능하지만 전원이 꺼지면 내용이 사라집니다.
  • ROM: 전원이 꺼져도 내용이 유지되며 기본 제어 정보 저장에 사용됩니다.
  • SSD, HDD: 파일과 프로그램을 장기 저장하는 보조기억장치입니다.
03

진법 변환

2진수, 8진수, 10진수, 16진수 사이의 변환은 자주 출제됩니다. 2진수 네 자리는 16진수 한 자리와 대응됩니다.

2진수 1010 = 10진수 10
2진수 1111 = 16진수 F
16진수 2A = 2진수 0010 1010
04

자료 표현

컴퓨터는 모든 자료를 0과 1로 표현합니다. 문자 코드는 ASCII, Unicode처럼 문자를 숫자 값으로 대응시키는 규칙입니다.

단위

1 Byte는 8 bit입니다. KB, MB, GB 단위 계산 문제에서는 2의 거듭제곱 관계를 확인하세요.

05

운영체제

운영체제는 사용자와 하드웨어 사이에서 자원을 관리합니다. 프로세스, 기억장치, 파일, 입출력 장치를 효율적으로 제어하는 역할을 합니다.

06

CPU 명령 처리 사이클

CPU는 하나의 명령어를 처리하기 위해 인출(Fetch) → 해석(Decode) → 실행(Execute) → 저장(Write Back) 단계를 반복합니다. 이 과정을 명령 사이클(Instruction Cycle)이라고 합니다.

단계설명관련 레지스터
인출 (Fetch)PC가 가리키는 주소에서 명령어를 메모리로부터 가져옵니다.PC, MAR, MBR
해석 (Decode)제어장치가 명령어를 해석하여 수행할 동작을 결정합니다.IR
실행 (Execute)ALU가 산술·논리 연산을 수행하거나 데이터를 이동합니다.ACC, ALU
저장 (Write Back)연산 결과를 레지스터 또는 메모리에 기록합니다.MAR, MBR
주요 레지스터 정리
  • PC (Program Counter): 다음 실행할 명령어의 주소를 저장합니다.
  • IR (Instruction Register): 현재 실행 중인 명령어를 저장합니다.
  • MAR (Memory Address Register): 접근할 메모리 주소를 저장합니다.
  • MBR (Memory Buffer Register): 메모리에서 읽거나 쓸 데이터를 임시 저장합니다.
  • ACC (Accumulator): 연산 결과를 임시 저장합니다.
07

버스 시스템

버스(Bus)는 CPU, 메모리, 입출력 장치 사이에서 데이터를 전달하는 공유 통신 경로입니다. 기능에 따라 세 가지로 구분됩니다.

버스 종류전달 내용방향
데이터 버스실제 데이터(명령어, 값)양방향
주소 버스메모리 또는 I/O 장치의 주소단방향 (CPU → 장치)
제어 버스읽기/쓰기, 인터럽트 등 제어 신호양방향
버스 폭과 성능

데이터 버스의 폭(비트 수)이 넓을수록 한 번에 전송 가능한 데이터 양이 늘어납니다. 주소 버스의 폭은 최대 접근 가능한 메모리 용량을 결정합니다 (32비트 주소 → 최대 4GB).

08

인터럽트 & 입출력 방식

인터럽트(Interrupt)는 CPU가 현재 작업을 중단하고 긴급한 요청을 처리하도록 하는 신호입니다. 처리 후 원래 작업으로 복귀합니다.

인터럽트 종류발생 원인
외부 인터럽트키보드 입력, 타이머, 전원 이상 등 외부 장치 신호
내부 인터럽트0으로 나누기, 오버플로우 등 CPU 내부 오류
소프트웨어 인터럽트프로그램의 명시적 인터럽트 명령 (SVC 등)

입출력 방식은 CPU 개입 정도에 따라 구분됩니다.

방식특징
프로그램 I/O (폴링)CPU가 장치 상태를 반복적으로 확인합니다. CPU 자원을 많이 소모합니다.
인터럽트 구동 I/O장치가 준비되면 인터럽트로 CPU에 알립니다. CPU 효율이 높습니다.
DMA (Direct Memory Access)CPU 개입 없이 메모리와 장치가 직접 데이터를 주고받습니다. 대용량 전송에 유리합니다.
09

기억장치 계층 구조

기억장치는 속도와 용량, 비용을 기준으로 계층을 형성합니다. 상위 계층일수록 빠르고 용량이 작으며 비쌉니다.

계층종류속도용량휘발성
1계층레지스터가장 빠름매우 작음 (수십 바이트)휘발성
2계층캐시 메모리 (L1/L2/L3)매우 빠름KB ~ 수십 MB휘발성
3계층주기억장치 (RAM)빠름GB 단위휘발성
4계층보조기억장치 (SSD/HDD)느림TB 단위비휘발성
캐시 적중률 (Hit Ratio)

CPU가 필요한 데이터를 캐시에서 찾는 비율을 말합니다. 적중률이 높을수록 평균 메모리 접근 시간이 줄어듭니다.

평균 접근 시간 = 캐시 접근 시간 × 적중률 + 주기억장치 접근 시간 × (1 - 적중률)
10

논리 회로 & 불 대수

논리 회로는 0과 1의 논리값을 처리하는 전자 회로입니다. 불 대수(Boolean Algebra)를 이용해 논리 연산을 수식으로 표현합니다.

게이트기호설명진리표 요약
ANDA · B두 입력이 모두 1일 때만 출력 11·1=1, 나머지=0
ORA + B하나 이상 1이면 출력 10+0=0, 나머지=1
NOTĀ입력을 반전0→1, 1→0
NAND̄(A·B)AND의 반전1·1=0, 나머지=1
NOR̄(A+B)OR의 반전0+0=1, 나머지=0
XORA ⊕ B두 입력이 다를 때 출력 10⊕1=1, 1⊕0=1, 같으면=0
드모르간 법칙 (자주 출제!) 
̄(A · B) = Ā + B̄    (NAND → NOT OR)
̄(A + B) = Ā · B̄    (NOR  → NOT AND)
불 대수 기본 법칙 
A + 0 = A       A · 1 = A       (항등 법칙)
A + 1 = 1       A · 0 = 0       (지배 법칙)
A + A = A       A · A = A       (멱등 법칙)
A + Ā = 1       A · Ā = 0       (보수 법칙)
11

보조기억장치 종류

보조기억장치는 전원이 꺼져도 데이터를 유지하는 비휘발성 저장 장치입니다. 저장 방식에 따라 특성이 크게 다릅니다.

종류저장 방식특징
HDD (하드디스크)자기 디스크대용량, 저렴, 속도 느림, 충격에 취약. 플래터·헤드·스핀들 구조.
SSD (솔리드 스테이트 드라이브)플래시 메모리빠른 속도, 충격에 강함, 소음 없음, HDD보다 비쌈.
USB 플래시 드라이브플래시 메모리휴대성 우수, 읽기/쓰기 횟수 제한 있음.
광학 디스크 (CD/DVD/BD)레이저 기록CD 700MB, DVD 4.7GB, Blu-ray 25GB 이상. 배포용 미디어로 사용.
NAS (Network Attached Storage)네트워크 연결네트워크를 통해 여러 사용자가 공유하는 스토리지.
HDD 주요 용어
  • 트랙(Track): 플래터 위의 동심원 경로
  • 섹터(Sector): 트랙을 분할한 최소 저장 단위 (보통 512 B)
  • 실린더(Cylinder): 여러 플래터에서 같은 위치의 트랙 집합
  • 탐색 시간(Seek Time): 헤드가 목표 트랙으로 이동하는 시간
  • 회전 지연(Rotational Latency): 목표 섹터가 헤드 아래에 올 때까지 기다리는 시간

명령 사이클

버스 시스템

데이터/주소/제어 버스의 방향과 기능 차이를 구분합니다.
