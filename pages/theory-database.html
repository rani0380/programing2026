테이블 구조, 키, SQL, 정규화, 트랜잭션처럼 시험에서 자주 확인하는 개념을 정리했습니다.

테이블과 스키마 키의 종류 SELECT 문 NULL 조건 정규화 트랜잭션
01

테이블과 스키마

관계형 데이터베이스는 데이터를 행과 열로 구성된 테이블에 저장합니다. 행은 레코드, 열은 속성 또는 필드라고 부릅니다.

02

키의 종류

키의미
기본키각 행을 유일하게 식별하는 속성입니다.
후보키기본키가 될 수 있는 유일성과 최소성을 가진 속성입니다.
외래키다른 테이블의 기본키를 참조해 관계를 연결합니다.
03

SELECT 문

SELECT 문은 테이블에서 필요한 데이터를 조회할 때 사용합니다. WHERE는 조건, ORDER BY는 정렬, GROUP BY는 그룹화를 담당합니다.

SELECT name, score
FROM student
WHERE score >= 60
ORDER BY score DESC;
04

NULL 조건

NULL은 값이 없거나 알 수 없음을 뜻합니다. 비교 연산자 대신 IS NULL, IS NOT NULL을 사용해야 합니다.

SELECT *
FROM student
WHERE phone IS NULL;
05

정규화

정규화는 데이터 중복과 이상 현상을 줄이기 위해 테이블을 구조적으로 나누는 과정입니다. 삽입, 삭제, 갱신 이상을 방지하는 목적이 있습니다.

시험 포인트

정규화의 목적은 중복 최소화와 데이터 무결성 향상입니다.

06

트랜잭션

트랜잭션은 하나의 논리적 작업 단위입니다. 모두 성공하면 COMMIT, 문제가 생기면 ROLLBACK으로 이전 상태로 되돌립니다.

원자성

작업은 전부 실행되거나 전부 실행되지 않아야 합니다.

일관성

처리 전후 데이터베이스 규칙이 유지됩니다.

고립성

동시에 실행되는 작업이 서로 간섭하지 않습니다.

지속성

완료된 결과는 영구적으로 반영됩니다.
