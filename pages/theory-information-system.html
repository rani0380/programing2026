01

네트워크

네트워크는 컴퓨터와 장치를 연결해 데이터를 주고받는 구조입니다. LAN은 가까운 범위, WAN은 넓은 범위의 통신망입니다. 네트워크는 유선·무선 방식으로 구분하며, 데이터는 패킷 단위로 전송됩니다.

용어의미
IP네트워크에서 장치를 식별하는 주소입니다.
TCP신뢰성 있는 연결형 전송을 제공합니다. 데이터 순서 보장, 오류 재전송 있음.
UDP빠른 비연결형 전송에 사용됩니다. 스트리밍·게임 등 속도 우선 환경에 적합.
LAN건물 내부 등 가까운 범위의 근거리 통신망입니다.
WAN도시·국가 간 연결하는 광역 통신망입니다.
DNS도메인 이름을 IP 주소로 변환하는 시스템입니다.
HTTP/HTTPS웹 데이터 전송 프로토콜. HTTPS는 SSL/TLS 암호화를 포함합니다.
OSI 7계층네트워크 통신을 7단계로 분리한 표준 모델입니다. (물리→데이터링크→네트워크→전송→세션→표현→응용)
🎯 기출 힌트

TCP와 UDP의 차이(신뢰성 vs 속도), OSI 7계층 순서, IP와 DNS 역할 구분이 자주 출제됩니다. TCP는 "연결형·신뢰성", UDP는 "비연결형·빠름"으로 암기하세요.

02

정보 보안

정보 보안의 핵심 목표는 기밀성, 무결성, 가용성입니다. 인증, 접근 제어, 암호화는 보안을 유지하기 위한 대표적인 방법입니다.

  • 기밀성(Confidentiality): 허가된 사용자만 정보를 볼 수 있습니다.
  • 무결성(Integrity): 정보가 임의로 변경되지 않도록 보장합니다.
  • 가용성(Availability): 필요한 때에 서비스를 사용할 수 있어야 합니다.
용어의미
암호화데이터를 읽을 수 없는 형태로 변환하여 기밀성을 보호합니다.
인증(Authentication)사용자가 본인임을 확인하는 절차입니다. (ID/PW, 생체인식 등)
접근 제어권한에 따라 자원 접근을 허용 또는 차단합니다.
방화벽(Firewall)허가되지 않은 네트워크 접근을 차단하는 보안 장치입니다.
백신(Antivirus)악성코드를 탐지하고 제거하는 프로그램입니다.
해시(Hash)데이터를 고정 길이의 값으로 변환. 무결성 검증에 사용됩니다.
공개키 암호화공개키로 암호화하고 개인키로 복호화하는 비대칭 암호 방식입니다.
SSL/TLS인터넷 통신을 암호화하는 보안 프로토콜입니다. HTTPS에 사용됩니다.
🎯 기출 힌트

보안 3요소(CIA)의 영문명과 의미를 함께 암기하세요. "기밀성=Confidentiality", "무결성=Integrity", "가용성=Availability". 암호화 방식(대칭/비대칭)과 해시 용도도 자주 출제됩니다.

03

소프트웨어 개발 단계

소프트웨어 개발은 요구사항 분석, 설계, 구현, 테스트, 유지보수의 흐름으로 진행됩니다. 각 단계의 산출물과 목적을 구분하세요.

단계주요 활동산출물
요구사항 분석사용자 요구 파악, 기능/비기능 요구사항 정의요구사항 명세서(SRS)
설계시스템 구조 설계, UI/DB 설계설계 명세서, 아키텍처 문서
구현(코딩)설계 기반으로 실제 코드 작성소스코드
테스트오류 발견 및 품질 검증테스트 결과 보고서
유지보수배포 후 버그 수정, 기능 개선수정된 소스코드, 패치
개발 방법론특징
폭포수(Waterfall)단계를 순차적으로 진행. 이전 단계로 돌아가기 어렵습니다.
애자일(Agile)짧은 반복(스프린트) 주기로 개발. 변경에 유연하게 대응합니다.
나선형(Spiral)위험 분석을 반복하며 점진적으로 개발합니다.
프로토타입시제품을 먼저 만들어 요구사항을 구체화합니다.
🎯 기출 힌트

개발 단계의 순서(분석→설계→구현→테스트→유지보수)와 각 단계의 산출물을 연결해서 암기하세요. 폭포수 모델이 가장 자주 출제되며, 애자일의 "반복적·유연성" 특징도 확인하세요.

04

테스트 기법

화이트박스 테스트는 내부 구조를 알고 검사하고, 블랙박스 테스트는 입력과 출력 중심으로 기능을 확인합니다.

구분화이트박스 테스트블랙박스 테스트
관점내부 구조(코드) 기반외부 기능(입출력) 기반
수행자개발자테스터(사용자 관점)
주요 기법구문 커버리지, 분기 커버리지동치 분할, 경계값 분석
테스트 종류범위설명
단위 테스트함수/모듈 단위가장 작은 단위의 코드를 독립적으로 검증합니다.
통합 테스트모듈 간 연동여러 모듈을 합쳐 인터페이스 오류를 검증합니다.
시스템 테스트전체 시스템완성된 시스템이 요구사항을 충족하는지 검증합니다.
인수 테스트사용자 환경최종 사용자가 실제 환경에서 수행하는 최종 검증입니다.
🎯 기출 힌트

테스트 종류는 범위가 좁은 것(단위)→넓은 것(인수) 순서로 암기하세요. 화이트박스="구조/내부/개발자", 블랙박스="기능/외부/사용자"로 구분합니다. 경계값 분석은 블랙박스 기법임을 기억하세요.

명령어WindowsLinux/Unix기능

06

기출 포인트

TCP와 UDP

TCP=연결형·신뢰성·순서보장, UDP=비연결형·빠름·스트리밍. 둘의 차이를 비교해서 암기하세요.

보안 3요소(CIA)

기밀성(C)·무결성(I)·가용성(A)의 영문 약어와 의미를 세트로 암기하세요.

개발 절차

분석→설계→구현→테스트→유지보수 순서와 각 단계의 산출물(SRS, 설계서, 소스코드)을 연결하세요.

테스트 종류

화이트박스(내부·구조)↔블랙박스(외부·기능), 단위→통합→시스템→인수 순서로 범위가 확장됩니다.

OS 명령어 쌍

Windows↔Linux 명령어 쌍(dir↔ls, del↔rm, copy↔cp)을 세트로 암기하세요.

OSI 7계층

물리-데이터링크-네트워크-전송-세션-표현-응용 순서를 암기하고, 각 계층의 대표 프로토콜을 연결하세요.
