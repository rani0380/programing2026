01

네트워크

네트워크는 컴퓨터와 장치들이 연결되어 데이터를 주고받는 통신망입니다. 프로토콜은 통신 규칙, 패킷은 데이터를 작게 나눈 전송 단위이며, 서버는 서비스를 제공하고 클라이언트는 서비스를 요청합니다.

용어의미
프로토콜통신을 하기 위한 규칙 또는 약속입니다.
패킷데이터를 작게 나눈 전송 단위입니다.
서버웹, 파일, 인증처럼 서비스를 제공하는 컴퓨터입니다.
클라이언트서버에 서비스를 요청하는 컴퓨터나 프로그램입니다.
호스트네트워크에 연결된 컴퓨터나 장치입니다.
대역폭한 번에 전송할 수 있는 데이터의 양입니다.
트래픽네트워크에서 이동하는 데이터의 양입니다.

LAN, WAN, 인터넷

구분범위예시
LAN가까운 범위의 근거리 통신망입니다.집, 학교, 컴퓨터실, 사무실
WAN지역이나 국가 간을 연결하는 넓은 범위의 통신망입니다.지역 간 통신망, 국가 간 통신망
인터넷전 세계 네트워크가 서로 연결된 거대한 통신망입니다.웹, 이메일, 검색 서비스

OSI 7계층과 TCP/IP 4계층

OSI 7계층은 네트워크 통신 과정을 7단계로 나눈 이론 모델이고, TCP/IP 4계층은 실제 인터넷에서 사용하는 통신 모델입니다. OSI 7계층은 물데네전세표응 순서로 암기합니다.

OSI 계층핵심 역할대표 프로토콜/장비
7 응용 계층사용자 서비스 제공HTTP, FTP, SMTP, DNS
6 표현 계층데이터 변환과 암호화JPEG, SSL/TLS
5 세션 계층연결 설정, 유지, 종료세션 관리
4 전송 계층신뢰성 있는 전송TCP, UDP
3 네트워크 계층경로 선택과 IP 주소 처리IP, Router
2 데이터링크 계층MAC 주소와 프레임 처리Switch, Ethernet
1 물리 계층전기 신호와 케이블 전송Hub, Cable
TCP/IP 계층OSI 대응대표 예
응용 계층OSI 5~7계층HTTP, HTTPS, FTP, SMTP, DNS
전송 계층OSI 4계층TCP, UDP
인터넷 계층OSI 3계층IP, ICMP

IP 주소, 서브넷, MAC 주소

IP 주소는 네트워크에서 장치를 구분하는 논리적 주소입니다. IPv4는 32비트이며 8비트씩 4개로 나누어 표기합니다. MAC 주소는 네트워크 장비에 부여된 48비트 물리적 주소입니다.

항목핵심 내용
공인 IP인터넷에서 직접 사용하는 전 세계 유일한 주소입니다. ISP가 할당합니다.
사설 IP내부 네트워크에서 사용하며 외부에서는 직접 접근할 수 없습니다. NAT를 통해 인터넷에 연결합니다.
사설 IP 범위10.x.x.x, 172.16~31.x.x, 192.168.x.x
서브넷 마스크IP 주소에서 네트워크 부분과 호스트 부분을 구분하는 값입니다.
/24255.255.255.0이며, 예를 들어 192.168.0.x가 같은 네트워크가 됩니다.
MAC 주소OSI 2계층에서 사용하는 하드웨어 고유 주소입니다. 스위치가 MAC 주소를 보고 전송합니다.

시험 포인트

DNS는 도메인을 IP로 변환, DHCP는 IP 자동 할당, NAT는 사설 IP와 공인 IP 변환입니다. TCP는 정확하게, UDP는 빠르게, HTTP=80, HTTPS=443, SSH=22, DNS=53을 우선 암기하세요.

02

정보 보안

정보 보안의 핵심 목표는 기밀성, 무결성, 가용성입니다. 인증, 접근 제어, 암호화는 보안을 유지하기 위한 대표적인 방법입니다.

  • 기밀성: 허가된 사용자만 정보를 볼 수 있습니다.
  • 무결성: 정보가 임의로 변경되지 않도록 보장합니다.
  • 가용성: 필요한 때에 서비스를 사용할 수 있어야 합니다.
03

소프트웨어 개발 단계

소프트웨어 개발은 요구사항 분석, 설계, 구현, 테스트, 유지보수의 흐름으로 진행됩니다. 각 단계의 산출물과 목적을 구분하세요.

04

테스트 기법

화이트박스 테스트는 내부 구조를 알고 검사하고, 블랙박스 테스트는 입력과 출력 중심으로 기능을 확인합니다.

자주 나오는 구분

단위 테스트, 통합 테스트, 시스템 테스트, 인수 테스트는 범위가 점점 넓어지는 흐름으로 이해하면 쉽습니다.

05

운영체제 명령어

명령어 문제는 파일과 디렉터리 조작, 현재 위치 확인, 목록 보기처럼 기본 동작을 묻는 경우가 많습니다.
